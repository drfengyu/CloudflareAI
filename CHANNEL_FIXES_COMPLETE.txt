## 🐛 问题列表

### 问题 1: 渠道详情页无法加载

**症状**:
- 点击渠道管理表单的某个渠道详情时页面跳转失败
- 提示"无法加载该页"

**原因**:
- 页面路由: `app/(dashboard)/admin/channels/[id]/page.tsx`
- API 路由: `app/api/channels/[channelId]/*`
- **路由参数不匹配**: `id` vs `channelId`

**修复**:
1. 重命名目录: `[id]` → `[channelId]`
2. 更新参数定义: `params: Promise<{ id: string }>` → `params: Promise<{ channelId: string }>`
3. 替换所有使用: `const { id } = await params` → `const { channelId } = await params`
4. 更新 API 调用路径: `/api/channels/${id}/` → `/api/channels/${channelId}/`

### 问题 2: 模型库显示"第三方"而非渠道名

**症状**:
- Vercel 渠道的模型在模型库界面显示为"第三方"
- 应该显示"Vercel"或自定义渠道名称

**原因**:
- `channel-catalog.ts` 第 98 行: `"openai-compatible": "第三方"`
- `model.author` 被设置为"第三方"，导致显示不友好

**修复**:
1. `fetchChannelModels()` 新增 `channelName` 参数
2. 使用 `channelName || channelLabel(channelType)` 作为 `model.author`
3. `fetchAllChannelsModels()` 传递 `c.name` 作为 label
4. `model-browser.tsx` 针对 `openai-compatible` 显示 `model.author`

### 问题 3: 模型图标显示"D"

**症状**:
- 所有渠道模型的图标都显示首字母"D"
- 因为 author 是"第三方"，首字母是"D"

**原因**:
- `VendorLogo` 组件使用 `author.charAt(0)` 作为首字母
- "第三方" → "D"

**修复**:
- 同问题 2，使用渠道名称作为 author
- "Vercel" → "V"

### 问题 4: Key 界面模型白名单为空

**症状**:
- 选择 Vercel 渠道后，模型白名单下拉框为空
- 无法选择模型

**原因**:
- `keys/page.tsx` 从 `model_pricing` 表查询模型
- 但没有过滤 `channelId IS NULL` 的记录
- 导致查询包含了所有模型，但渠道过滤时匹配不到

**修复**:
```typescript
// 旧代码
const pricingModels = await db
  .select({ modelId: modelPricing.modelId, channelId: modelPricing.channelId })
  .from(modelPricing);

// 新代码
const pricingModels = await db
  .select({ modelId: modelPricing.modelId, channelId: modelPricing.channelId })
  .from(modelPricing)
  .where(sql`${modelPricing.channelId} IS NOT NULL`);
```
